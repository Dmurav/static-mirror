'монтирование к текущей директории директории в контейнере'

1. создаём докер контейнер на базе ubuntu и запускаем его, монтируя директорию хоста к директории контейнера
2. внутри контейнера создаём скрипт для загрузки зеркала сайта в смонтированную директорию
3. внутри контейнера настраиваем crontab для запуска bash скрипта по расписанию
4. записываем итоговый докер-образ
5. создаём файл docker-compose, который будет основой для деплоя
6. на хостовой машине устанавливаем докер
7. создаём makefile, который будет реализовывать api для управления докер-композ файлом

docker-compose build docker-compose.yml  - билдит и деплоит докер контейнер
docker-compose up - выполняется команда RUN докерфайла
docker-compose down - остановить и удалить контейнер
docker-compose logs -f server - посмотреть логи запущенного сервиса
docker-compose ps - вывести список контейнеров
docker-compose exec [service name] [command] - выполнить команду в запущенном контейнере
docker-compose images - вывести список всех образов
